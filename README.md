## Скрипт для пакетной оптимизации и сжатия коллекций книг FB2.

### Что умеет:
 - Сжатие картинок - преобразование в JPEG с указанным качеством сжатия
 - Уменьшение картинок до указанной ширины с сохранением пропорций
 - Удаление дубликатов картинок, с соответствующим переименованием ссылок
 - Удаление неиспользуемых картинок
 - Удаление битых картинок (не удалось прочитать base64, или данные не интерпретируются как изображение)
 - Смена кодировки файлов с UTF-8 на windows-1251 для уменьшения размера файла
 - Оптимизация XML тэгов, удаление лишних пробелов (текст при этом не затрагивается)
 - Чтение напрямую из zip-архивов
 - Сохранение в zip-архивы
 - Прочие мелочи

### Эффективность функций
 - Первое место - сжатие картинок, дает максимальный эффект
 - Второе место - удаление дублированных, неиспользуемых, поврежденных и безымянных картинок
 - Третье место - перекодирование из UTF-8 в windows-2151, особенно эффективно если у вас мало книг с картинками.
 - Четвертое место - очистка переносов и пробелов между тэгами XML, минимальный эффект.

### Как пользоваться:
  1. Необходимо установить на компьютер интерпретатор PHP ( https://php.net )
  2. Запустить из командной строки: `php fb2.optimize.php --параметр1 --параметр2 ...`

### Параметры командной строки:
  **`--src, -s`**				- **обязательный** параметр. Папка с исходными книгами (файлы .fb2 или .zip архивы содержащие файлы .fb2).
													Книги могут находиться как в корне папки, так и во вложенных папках, для скрипта это не проблема.

  **`--dest, -d`**			- **обязательный** параметр. Папка назначения, в которую будут сохранены обработанные книги.
													Папку заранее создавать необязательно, скрипт попытается сделать это за Вас.

  **`--jpeg, -j`**			- число от 0 до 100. Качество сжатия JPEG. 
												- Если указано, то все картинки будут преобразованы в JPEG с выбранным качеством.
													Если не указывать или указать ноль, то картинки будут пережиматься с качеством по умолчанию 75%, 
													но только если указан параметр `--width` и картинка действительно подлежит ресайзу.

  **`--width, -w`**			- число. Максимальная ширина картинки в пикселях.
													Изображения будут уменьшены по ширине до указанного размера, с сохранением соотношения сторон.
													Картинки меньше указанной ширины будут проигнорированы, если преобразование в JPEG не указано.
  
  **`--overwrite, -o`**	- флаг. Перезаписывать файлы в папке назначения.
  
  **`--convert, -c`**		- флаг. Конвертировать файл из utf-8 в windows-1251 или нет. 
													Успешное перекодирование не гарантируется, в этом случае кодировка файла останется без изменений. 
													Если процент перекодированных файлов нужно повысить, используйте небезопасное перекодирование ( см. `--unsafe` ).
  
  **`--unsafe, -u`**		- флаг. **Небезопасное преобразование кодировки.** Используется только вместе с `--convert`.
													Скрипт будет пытаться транслитерировать юникод при конвертации в windows-1251, только если безопасная конвертация не удалась. 
													С этим флагом есть риск потерять или исказить некоторые символы. 
													Не рекомендуется к использованию, т.к. практика показывает что с этим флагом общий объем библиотеки будет ненамного меньше, чем без.
													Если флаг не указан, то будет применяться безопасное перекодирование без потери символов.
  
  **`--fix, -f`**				- флаг. Оптимизировать код XML - уборка лишних пробелов и переносов строк. 
  												Затрагиваются только тэги, текст не оптимизируется.
  
  **`--zip, -z`**				- флаг. Каждая книга будет упакована в отдельный архив.

#### Примеры использования для случая обработки книг из папки `D:\LibRusEc` с сохранением в папку `D:\LibEusEC-compressed`:

+ Приемлемое соотношение размер-качество (уменьшение библиотеки примерно на 65%):<br>
`php fb2.optimize.php --src="D:\LibRusEc" --dest="D:\LibEusEC-compressed" --width=500 --jpeg=50 --overwrite --convert --fix --zip`

+ То-же самое с сокращёнными параметрами:<br>
`php fb2.optimize.php -s"D:\LibRusEc" -d"D:\LibEusEC-compressed" -w500 -j50 -ocfz`

+ Экстремальное сжатие с плохим качеством картинок и с риском потери некоторых юникод-символов:<br>
`php fb2.optimize.php --src="D:\LibRusEc" --dest="D:\LibEusEC-compressed" --width=300 --jpeg=30 --overwrite --convert --unsafe --fix --zip`

+ Простая оптимизация без замены кодировки, без упаковки в zip, без преобразования изображений (но дубликаты и неиспользуемые картинки все равно будут очищены):<br>
`php fb2.optimize.php --src="D:\LibRusEc" --dest="D:\LibEusEC-compressed"`
